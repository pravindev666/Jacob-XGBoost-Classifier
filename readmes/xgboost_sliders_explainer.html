The 4 XGBoost sliders — what they actually mean
n_estimators = 300
Number of trees in the forest
300 trees
1003005007501000
XGBoost builds many small decision trees and combines them. Think of it like asking 300 different analysts "will Nifty go up?" and taking the majority vote. More trees = more opinions = better accuracy, but also slower to train.
Analogy: You ask 10 friends for restaurant advice → decent answer. You ask 300 friends → much better answer. But at 1000 friends, you're wasting time because they all start repeating each other.
Recommended: 300–500. Below 100 = underpowered. Above 500 = slow with minimal gain unless you have huge data.
max_depth = 5
How deep each tree can grow
depth 5
345678910
Each tree asks questions: "Is RSI below 30? → Is VIX above 20? → Is FII buying? → ..." Max depth = how many questions it can ask. Depth 5 means 5 levels of questions before making a prediction.
Too shallow (depth 2-3): "Is RSI below 30? → YES = predict UP" — too simple, misses patterns.
Too deep (depth 8-10): "Is RSI 28.47 AND VIX exactly 19.83 AND it's a Tuesday in March AND..." — too specific, only works on training data (overfitting).
Depth 5: Goldilocks — complex enough to find real patterns, simple enough to generalise.
High depth = OVERFITTING danger. If train accuracy is 80% but val accuracy is 55%, reduce max_depth first.
learning_rate = 0.05
How fast the model learns from each tree
slow & steady
0.010.050.100.20
After each tree, the model corrects its mistakes. Learning rate = how big a correction it makes. Small correction = learns slowly but more accurately. Big correction = learns fast but overshoots.
Analogy: Learning to ride a bike. Small adjustments (0.01) = very slow learning but perfect balance. Big jerky adjustments (0.2) = quick to try but keeps falling. 0.05 = confident but controlled.
Rule: Lower learning_rate → increase n_estimators. If lr=0.01, use 1000 trees. If lr=0.1, use 200 trees. They trade off.
subsample = 0.80
What % of data each tree sees
80% of data
0.50.60.70.80.91.0
Each tree is trained on a random 80% sample of your data. The 20% it didn't see keeps it honest — it can't memorise every single data point. This is called "bagging" and it's one of the main reasons XGBoost doesn't overfit as badly.
Analogy: Studying for an exam with 1000 practice questions. Each time you study, you randomly pick 800. By the end, you've seen all questions many times but from different angles — you understand concepts, not just memorised answers.
Keep at 0.8. Going below 0.6 introduces too much randomness. Keeping at 1.0 (all data) = more overfit risk.
The 2 year dropdowns — Train until & Validate until
Train until year = 2021
Everything from 2015 to end of 2021 is used to teach the model. This is ~1,500 trading days. The model reads every day's features and learns which patterns predict next-day direction.

Why not 2022 or 2023? Because you need some data the model has NEVER seen to measure real accuracy. If you train until yesterday, the test set is empty — you have no way to verify if it actually works.
Validate until year = 2023
2022–2023 data is used DURING training — not to learn patterns, but to check if the model is overfitting. XGBoost's early stopping watches this validation score. When val accuracy stops improving for 30 rounds in a row → stop training. This prevents memorisation.

Think of it as: A teacher checking your homework while you study. Not the final exam — just a check to make sure you're learning, not copying.
What's left = automatic test set
Everything after "Validate until 2023" becomes the test set (2024–2025). The model has NEVER seen this. The accuracy shown on the test set is the real-world accuracy you should expect in live trading. This is your ground truth.
